1         Introduction

1.1              Overview

DirectAssure 2.0 allows providers to confirm their practice locations to plan-specific practice location information. This DirectAssure 2.0 extract specifications document is a compilation of the data elements, layouts, formats and procedures for Participating Organizations to consume the DirectAssure 2.0 extract. The DirectAssure 2.0 extract is available through an API described in the following sections. The output of the extract will be available as a JSON file. (JSON is a lightweight and simple way to represent machine-readable data).

1.2              Purpose

This document is intended to serve as a reference for data layout, field definitions, naming standards and all other information pertinent to DirectAssure 2.0 extract data. Please refer to the CAQH DirectAssure 2.0 Practice Location Guidelines document for DirectAssure 2.0 input file specifications.

1.3              Target Audience

This document is intended for business and technical teams (data stewards, developers, operational and technical staff etc.) at the Participating Organizations involved in the exchange of data between CAQH DirectAssure and their internal systems.

2         DirectAssure 2.0 Extract Service

DirectAssure 2.0 Extract service can be used by Participating Organizations to access provider practice location data from CAQH ProView. The DirectAssure 2.0 API is designed to accept a range of attestation dates and return relevant Provider Directory data elements for all providers who attested within that date range. The extract can be requested for a date range that spans seven days or less. If the date range exceeds seven days, an error code will be returned. If there are multiple attestations for a provider within a date range, only the latest attestation data within the date range will be returned. The service will return data only for providers who have attested after DirectAssure 2.0 go-live date.

The extract will include the version of the address submitted by the plan in their input file plan_submitted_practice_locations (if applicable), the standardized version of the address standardized_practice_location (if applicable), and the address the provider attested to display_practice_location. The display address will contain the providers response.

The extract API will be available to all Participating Organizations enrolled in the DirectAssure product (including those signed up for DirectAssure 1.0). DirectAssure 2.0 allows Participating Organization to validate their practice location accuracy by comparing it to provider-supplied responses by submitting an input file. (Please refer to the CAQH DirectAssure 2.0 Practice Location Guidelines document for more details). The extract API can be requested without an input file submission, however, certain sections pertaining to input file submission (e.g.: plan_submitted_practice_locations) in the extract will be blank.

 

The DirectAssure 2.0 API service will return Provider data only if ALL the following criteria are met:

·        Provider Status must be Reattestation or Initial Profile Complete.

·        Provider has authorized the Participating Organization to see their information.

·        Participating Organization roster status of the Provider is active.

·        Provider has confirmed his/her Directory Information.

provider_address_response_code

Code identifying providers response to the practice location address displayed on the portal for verification. Valid values are Reviewed Address, Rejected Address, Accepted Address, Provider Created Address and Archived Address.

Response code of Reviewed Address means the provider was presented with the address and did not accept or reject, but clicked on I dont know option in the ProView application.

Response code of Rejected Address means that the provider rejected the practice location address presented to them.

Response code of Accepted Address means the provider accepted the suggested practice location address and completed the necessary actions required to attest their profile.

Response code of Provider Created Address means the provider already had this address in their profile.

Response code of Archived Address means the provider is no longer practicing at this address.

Code identifying the reason for a providers response to the practice location address. The reason code will ONLY be available for response codes of Rejected Address and Archived Address.

Reason code of Provider Edited Address implies the provider edited one or more address fields in this previously attested to record, therefore archiving it.

Reason code of Address is incorrect is a value the provider selected as a reason for rejection when presented with this address for reconciliation.

Reason code of No longer practice here is a value the provider selected as a reason for rejection when presented with this address for reconciliation or as the reason for archive when archiving a previously attested to location.

Reason code of Never practiced here, location is part of my group is a value the provider selected as a reason for rejection when presented with this address for reconciliation or as the reason for archive when archiving a previously attested to location.

Reason code of Never practiced here, entered by mistake is a value the provider selected as the reason for archive when archiving a previously attested to location.

Reason code of No Reason Specified is a value populated by the system for all provider responses during the alpha and beta phase of DirectAssure 2.0 (The reason code was not available at the time)
